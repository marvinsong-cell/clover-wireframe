## 1. 배경 및 목표

현재 어드민은 회원 가입/승인 관리 중심으로만 운영되고 있음. 서비스가 본격 운영 단계에 진입하면서 아래 운영 업무가 수작업으로 처리되고 있어 효율화가 필요함.

| 현재 문제 | 해결 목표 |
|---|---|
| 입금 여부를 별도로 관리 | 어드민 내 입금 정보 통합 관리 |
| 매니저 구분 없이 전체 회원 혼재 | 매니저별 회원 필터링으로 독립 운영 |
| 매칭용 프로필을 개별 조회해야 함 | 카드뷰로 한 눈에 비교 가능하게 |
| 반려 처리 시 실수 방지 장치 없음 | 반려 확인 모달로 이중 확인 |

## 2. 회원 목록 — 탭 구조 개편

### 2-1. 개요

기존 어드민의 탭 구조(심사대기 / 승인 / 반려 / 전체)를 **그대로 유지**하면서, 각 탭에 검색·필터·신규 컬럼을 추가한다.

### 2-2. 탭별 구성

| 탭 | 대상 회원 | 서브 필터 칩 | 주요 추가 기능 |
|---|---|---|---|
| **심사대기** | 가입 신청 대기 중인 회원 | — | 승인 / 반려 버튼 |
| **승인** | 승인 완료된 회원 | 전체 / 미발송 / 발송완료 | 입금여부·입금액·담당매니저·프로필 컬럼, 추가 필터 |
| **반려** | 반려 처리된 회원 | — | 반려일 컬럼 |
| **전체** | 모든 회원 | 심사대기·승인·반려 | 상태 배지 |

### 2-3. 공통 기능 (모든 탭)

- **이름 / 연락처 검색창**
- **성별 필터** (전체 / 남성 / 여성)
- **엑셀 다운로드** 버튼

### 2-4. 승인 탭 추가 필터

승인 탭에서만 노출되는 추가 필터:

- **입금 여부** (전체 / 입금완료 / 미입금)
- **담당매니저** (전체 / 김세연 / 이영현 / 송규용)

### 2-5. 탭별 테이블 컬럼

**심사대기 탭**

|---|---|
| □ 체크박스 | 일괄 선택 |
| 이름 / 성별 / 나이 / 거주지 / 학교 / 직장·직무 / 연락처 / 가입일 | 기존 컬럼 |
| 관리 | 승인 버튼 (초록) + 반려 버튼 (빨강) |

**승인 탭** ← 신규 컬럼 추가

| 입금 여부 | 입금완료(초록) / 미입금(빨강) 배지 ← **신규** |

| 프로필 | 완료 / 미완료 배지 ← **신규** |
| 문자 발송 | 발송완료(파란색) / 미발송 — 기존 컬럼 |
| 관리 | 상세 버튼 |

**반려 탭**

| 반려일 | 반려 처리 날짜 ← **신규** |
| 관리 | 상세 버튼 |

**전체 탭**

## 3. 기능 1 — 입금 관리

### 3-1. 개요

회원별 입금 여부와 입금액을 어드민에서 직접 기입하고, 전체 월별 수익을 대시보드에서 조회할 수 있도록 한다.

### 3-2. 가격 정책

| 유형 | 금액 | 비고 |
|---|---|---|
| 정가 | 39,000원 | 기본가 (고정) |
| 할인가 | 직접 입력 | 운영자 재량 적용, 금액 자유 입력 |

### 3-3. 회원 상세 — 입금 정보 패널

회원 상세 페이지에 독립된 **입금 정보** 패널을 추가한다.

**입력 항목:**

- **가격 유형** (라디오 선택)
  - 정가: 39,000원 고정 표시
  - 할인가: 받은 금액을 숫자로 직접 입력
- **입금일** (날짜 선택)
- **입금 상태** (드롭다운): 입금완료 / 미입금

**저장 시 동작:** 해당 회원 레코드에 저장, 수익 집계에 자동 반영

### 3-4. 입금·수익 현황 페이지 (신규)

사이드바에 **입금·수익 현황** 메뉴를 신설한다.

**상단 지표 카드 (3개):**

| 지표 | 설명 |
|---|---|
| 이달 총 입금액 | 해당 월 전체 입금 합산 |
| 이달 순수익 | 총 입금액 기준 |
| 누적 수익 | 연간 누적 |

**월별 수익 추이 차트:** 연도 선택 → 월별 바 차트

**구성 현황:** 정가 / 할인가 비율 바 형태 표시

**입금 내역 리스트:** 회원명 / 날짜 / 유형 / 금액, 월별 필터

## 4. 기능 2 — 담당매니저 배정

### 4-1. 개요

각 회원에게 담당 매니저를 배정하여, 매니저가 어드민에 로그인했을 때 자신이 담당하는 회원만 조회할 수 있게 한다.

**현재 매니저 (3명):** 김세연 / 이영현 / 송규용

### 4-2. 회원 상세 — 담당매니저 배정 UI

회원 정보 패널 하단에 매니저 칩 선택 UI를 배치한다.

- 매니저 목록(김세연 / 이영현 / 송규용)을 칩 형태로 나열
- 선택된 매니저는 강조 표시
- 변경 즉시 저장 (별도 저장 버튼 없음)

## 5. 기능 3 — 프로필 카드뷰

### 5-1. 개요

온보딩(프로필 작성)을 완료한 회원을 카드 형태로 한 번에 조회하여, 매칭 대상을 시각적으로 비교할 수 있도록 한다. **조회 전용이며, 프로필 전달 및 매칭 진행은 수동으로 별도 처리한다.**

### 5-2. 진입 경로

- 사이드바 **프로필 카드뷰** 메뉴
- 승인 탭 우측 상단 **[프로필 카드뷰]** 버튼

### 5-3. 필터

| 필터 | 옵션 |
|---|---|
| 성별 | 전체 / 남성 / 여성 |
| 나이대 | 전체 / 20대 / 30대 |
| 담당매니저 | 전체 / 김세연 / 이영현 / 송규용 |

### 5-4. 카드 구성

**카드 헤더:** 프로필 사진 (없을 경우 이니셜 아바타) + 성별 배지

**카드 본문:** 이름 / 나이·지역·직업 / 주요 조건 칩 (키·학력·흡연 여부)

**카드 하단:** [상세보기] → 회원 상세 페이지로 이동

## 6. 기능 4 — 반려 확인 모달

### 6-1. 개요

심사대기 회원의 가입 신청을 반려할 때, 실수로 버튼을 누르는 것을 방지하기 위해 확인 모달을 노출한다.

### 6-2. 노출 위치

- 회원 목록 **심사대기 탭** → 각 회원 행의 **[반려]** 버튼 클릭 시
- **대시보드** → 승인 대기 패널의 **[반려]** 버튼 클릭 시

### 6-3. 모달 구성

| 요소 | 내용 |
|---|---|
| 제목 | 반려 처리 확인 |
| 본문 | "{회원명}님의 가입 신청을 반려합니다. 반려 후에는 회원이 재신청해야 합니다." |
| 버튼 | 취소 / 반려 확인 (빨간색) |

**[취소]** 클릭 시: 모달 닫힘, 아무 동작 없음  
**[반려 확인]** 클릭 시: 해당 회원 상태를 반려로 변경, 반려 탭으로 이동

## 7. 정보 구조(IA) 변경사항

```
Clover Admin
├── 대시보드
│   └── 미입금 현황, 승인 대기 회원 (반려 버튼 포함)
│
├── 회원 관리
│   ├── 회원 목록
│   │   ├── [심사대기 탭]  ← 승인/반려 버튼
│   │   ├── [승인 탭]      ← 입금여부·입금액·담당매니저·프로필 컬럼 추가
│   │   │                     입금·담당매니저 필터, 검색, 엑셀 다운로드
│   │   ├── [반려 탭]      ← 반려일 컬럼
│   │   └── [전체 탭]      ← 상태 배지
│   │       └── 회원 상세  ← 입금 정보 패널, 담당매니저 배정 UI 추가
│   └── 프로필 카드뷰      ← 신규 (조회 전용)
│
├── 정산 / 수익
│   └── 입금·수익 현황     ← 신규
│
└── 운영
    └── 설정
```
