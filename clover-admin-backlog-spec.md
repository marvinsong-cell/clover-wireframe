## 1. 배경 및 목표

현재 어드민은 회원 가입/승인 관리 중심으로만 운영되고 있음. 서비스가 본격 운영 단계에 진입하면서 아래 운영 업무가 수작업으로 처리되고 있어 효율화가 필요함.

| 현재 문제 | 해결 목표 |
|---|---|
| 입금 여부를 별도 스프레드시트로 관리 | 어드민 내 입금 정보 통합 관리 |
| 매니저 구분 없이 전체 회원이 혼재 | 매니저별 회원 필터링으로 독립 운영 |
| 매칭용 프로필을 개별 조회해야 함 | 카드뷰로 한 눈에 비교 가능하게 |
| 매칭 승인/거절 결과를 메모에 직접 기입 | 승인/거절/약속을 구조화된 데이터로 기록 |

## 2. 기능 1 — 입금 관리

### 2-1. 개요

회원별 입금 여부, 입금 유형(정가/할인가), 입금액을 어드민에서 직접 기입하고, 전체 월별 수익/환불 내역을 대시보드에서 조회할 수 있도록 한다.

### 2-2. 가격 정책

| 유형 | 금액 | 비고 |
|---|---|---|
| 정가 | 2,000,000원 | 기본가 |
| 할인가 | 1,500,000원 | 운영자 재량 적용 |

### 2-3. 회원 목록 — 추가 컬럼

기존 회원 목록 테이블에 아래 컬럼을 추가한다.

| 컬럼명 | 내용 | 표시 방식 |
|---|---|---|
| 입금 여부 | 입금완료 / 미입금 / 환불 | 컬러 배지 |
| 입금액 | 숫자 + 유형 표시 | `150만원` + `할인가` 배지 |

**필터 추가:** 입금 여부(전체 / 입금완료 / 미입금 / 환불)

### 2-4. 회원 상세 — 입금 정보 섹션

회원 상세 페이지에 독립된 **입금 정보** 패널을 추가한다.

**입력 항목:**

- **가격 유형** (라디오 선택): 정가 2,000,000원 / 할인가 1,500,000원
- **입금액** (숫자 직접 입력, 원 단위)
- **입금일** (날짜 선택)
- **입금 상태** (드롭다운): 입금완료 / 미입금 / 환불 / 부분환불

**저장 시 동작:** 해당 회원 레코드에 저장, 수익 집계에 자동 반영

### 2-5. 입금·수익 현황 페이지 (신규)

사이드바에 **입금·수익 현황** 메뉴를 신설한다.

**상단 지표 카드 (4개):**

| 지표 | 설명 |
|---|---|
| 이달 총 입금액 | 해당 월 전체 입금 합산 |
| 이달 환불액 | 해당 월 환불 합산 (음수 표시) |
| 이달 순수익 | 총 입금액 - 환불액 |
| 누적 수익 | 연간 누적 |

**월별 수익 추이 차트:** 연도 선택 → 월별 바 차트 (Chart.js 연동 예정)

**구성 현황:** 정가 / 할인가 / 환불 비율 바 형태 표시

**입금 내역 리스트:** 회원명 / 날짜 / 유형 / 금액, 월별 필터 및 입금/환불 구분 필터

## 3. 기능 2 — 담당매니저 배정

### 3-1. 개요

각 회원에게 담당 매니저를 배정하여, 매니저가 어드민에 로그인했을 때 자신이 담당하는 회원만 조회할 수 있게 한다.

### 3-2. 회원 목록 — 추가 컬럼 및 필터

| 컬럼명 | 내용 |
|---|---|
| 담당매니저 | 배정된 매니저 이름 배지. 미배정 시 `[배정]` 버튼 노출 |

**필터 추가:** 담당매니저 드롭다운 (전체 / 김매니저 / 이매니저 / 박매니저 …)

매니저 계정으로 로그인 시 기본 필터가 본인 담당 회원으로 고정됨.

### 3-3. 회원 상세 — 담당매니저 배정 UI

회원 정보 패널 하단에 매니저 칩 선택 UI를 배치한다.

- 매니저 목록을 칩 형태로 나열
- 선택된 매니저는 강조 표시
- 변경 즉시 저장 (별도 저장 버튼 없음)

### 3-4. 매니저 관리 페이지 (신규)

사이드바에 **매니저 관리** 메뉴를 신설한다.

**조회 항목:**

| 항목 | 설명 |
|---|---|
| 매니저 이름 / 이메일 | 기본 정보 |
| 담당 회원 수 | 현재 배정된 회원 수 |
| 이달 매칭 수 | 해당 매니저가 진행한 매칭 건수 |
| 승인율 | 승인 / 전체 매칭 비율 |
| 입금 관리 금액 | 담당 회원의 총 입금액 합산 |

**[담당 회원보기] 버튼:** 해당 매니저 필터가 적용된 회원 목록으로 이동

## 4. 기능 3 — 프로필 카드뷰

### 4-1. 개요

온보딩(프로필 작성)을 완료한 회원을 카드 형태로 한 번에 조회하여, 매칭 대상을 시각적으로 비교하고 바로 매칭을 진행할 수 있도록 한다.

### 4-2. 진입 경로

- 사이드바 **프로필 카드뷰** 메뉴
- 회원 목록 상단 **[프로필 카드뷰]** 버튼

### 4-3. 필터

| 필터 | 옵션 |
|---|---|
| 성별 | 전체 / 남성 / 여성 |
| 나이대 | 전체 / 20대 / 30대 |
| 담당매니저 | 전체 / 개별 매니저 |

### 4-4. 카드 구성

각 카드에 아래 정보를 표시한다.

**카드 헤더:**
- 프로필 사진 (없을 경우 이니셜 아바타)
- 성별 배지 (남/여)

**카드 본문:**
- 이름
- 나이 · 지역 · 직업
- 주요 조건 칩: 키 / 학력 / 흡연 여부

**카드 하단 액션:**
- **[상세보기]** → 회원 상세 페이지 이동
- **[매칭]** → 해당 회원을 대상으로 매칭 추가 진행

## 5. 기능 4 — 매칭 승인/거절 및 약속 관리

### 5-1. 개요

매니저가 고객에게 상대방 프로필 카드를 전달한 후, 고객의 승인/거절 결과를 어드민에 기록한다. 승인 시 약속(장소/시간)을 등록하면 두 회원의 매칭 이력에 동시에 저장된다.

### 5-2. 매칭 추가 (회원 상세 → 우측 패널)

**입력 항목:**
- 매칭 상대 선택 (회원 드롭다운)
- 매칭 메모 (자유 입력)

**[프로필 카드 전달] 버튼:** 클릭 시 매칭 레코드 생성, 상태 = `검토중`

### 5-3. 매칭 결과 입력

**검토중 상태의 매칭에 결과를 입력한다.**

| 결과 | 후속 동작 |
|---|---|
| 승인 | 매칭 이력에 `승인` 기록, 약속 등록 가능 상태로 전환 |
| 거절 | 거절 사유 선택 화면 노출 |

**거절 사유 선택지 (복수 선택 가능):**
- 나이 차이
- 외모 미매칭
- 직업 미매칭
- 거리 문제
- 종교 차이
- 기타 (자유 입력)

### 5-4. 약속 등록 모달

승인된 매칭에서 **[약속]** 버튼 클릭 시 모달을 열어 아래 항목을 입력한다.

| 항목 | 형식 |
|---|---|
| 장소 | 텍스트 자유 입력 |
| 날짜 | 날짜 선택 |
| 시간 | 시간 선택 |
| 메모 | 자유 입력 (선택) |

**저장 동작:** 두 회원(A, B) 모두의 매칭 이력에 약속 정보가 동시에 저장됨. 저장 전 안내 문구 표시.

> "약속 정보를 저장하면 [회원 A]님과 [회원 B]님 두 분의 기록에 모두 저장됩니다."

### 5-5. 매칭 이력 표시 (회원 상세 → 운영 메모 상단)

운영 메모 섹션 상단에 **매칭 이력** 블록을 고정 노출한다.

**표시 형식:**

```
매칭 이력
─────────────────────────────────
● 김철수 [승인]                    [약속]
  2026.05.01 · 매칭 승인
  📍 강남 스타벅스 강남점 / 2026.05.15 19:00

● 이민호 [거절]
  2026.04.25 · 매칭 거절
  사유: 나이 차이, 직업 미매칭

● 박성준 [검토중]
  2026.05.05 · 프로필 전달 완료
─────────────────────────────────
일반 메모
[자유 입력 텍스트]
```

**매칭 상태별 표시:**

| 상태 | 색상 | 표시 내용 |
|---|---|---|
| 승인 | 초록 | 상대 이름, 약속 정보, [약속] 버튼 |
| 거절 | 빨강 | 상대 이름, 거절 사유 |
| 검토중 | 노랑 | 상대 이름, 전달 일시 |

### 5-6. 매칭 관리 페이지 (신규)

사이드바에 **매칭 관리** 메뉴를 신설한다.

**상단 통계:**
- 총 매칭 진행 / 승인 / 거절 / 검토중 건수

**테이블 컬럼:**
- 회원 A / 회원 B / 매칭일 / 결과 / 거절 사유 / 약속 일정 / 담당매니저 / 관리

**필터:** 결과(전체/승인/거절/검토중) / 담당매니저 / 약속 여부

## 6. 정보 구조(IA) 변경사항

```
Clover Admin
├── 대시보드
│   └── 미입금 현황, 최근 매칭 이력
│
├── 회원 관리
│   ├── 회원 목록          ← 입금 여부·입금액·담당매니저 컬럼 추가
│   │   └── 회원 상세      ← 입금 정보 섹션, 담당매니저 배정, 매칭 이력 추가
│   ├── 프로필 카드뷰      ← 신규
│   └── 매칭 관리          ← 신규
│
├── 정산 / 수익
│   └── 입금·수익 현황     ← 신규
│
└── 운영
    ├── 매니저 관리        ← 신규
    └── 설정
```

## 7. 데이터 모델 (필드 추가)

### members 테이블 추가 필드

| 필드명 | 타입 | 설명 |
|---|---|---|
| `payment_status` | enum | `UNPAID` / `PAID` / `REFUNDED` / `PARTIAL_REFUNDED` |
| `payment_type` | enum | `FULL_PRICE` / `DISCOUNTED` |
| `payment_amount` | integer | 실제 입금액 (원 단위) |
| `payment_date` | date | 입금일 |
| `manager_id` | uuid | 담당 매니저 FK |

### matches 테이블 (신규)

| 필드명 | 타입 | 설명 |
|---|---|---|
| `id` | uuid | PK |
| `member_a_id` | uuid | 회원 A FK |
| `member_b_id` | uuid | 회원 B FK |
| `status` | enum | `PENDING` / `APPROVED` / `REJECTED` |
| `rejection_reasons` | text[] | 거절 사유 배열 |
| `rejection_detail` | text | 거절 상세 메모 |
| `matched_by` | uuid | 매칭 진행 매니저 FK |
| `created_at` | timestamptz | 매칭 생성일 |

### appointments 테이블 (신규)

| 필드명 | 타입 | 설명 |
|---|---|---|
| `id` | uuid | PK |
| `match_id` | uuid | 매칭 FK |
| `location` | text | 약속 장소 |
| `appointment_at` | timestamptz | 약속 일시 |
| `memo` | text | 약속 메모 |
| `created_by` | uuid | 등록 매니저 FK |
| `updated_at` | timestamptz | 최종 수정일 |
