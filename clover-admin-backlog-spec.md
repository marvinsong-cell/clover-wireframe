## 1. 배경 및 목표

현재 어드민은 회원 가입/승인 관리 중심으로만 운영되고 있음. 서비스가 본격 운영 단계에 진입하면서 아래 운영 업무가 수작업으로 처리되고 있어 효율화가 필요함.

| 현재 문제 | 해결 목표 |
|---|---|
| 입금 여부를 별도로 관리 | 어드민 내 입금 정보 통합 관리 |
| 매니저 구분 없이 전체 회원 혼재 | 매니저별 회원 필터링으로 독립 운영 |
| 매칭용 프로필을 개별 조회해야 함 | 카드뷰로 한 눈에 비교 가능하게 |
| 반려 처리 시 실수 방지 장치 없음 | 반려 확인 모달로 이중 확인 |

## 2. 기능 1 — 입금 관리

### 2-1. 개요

회원별 입금 여부와 입금액을 어드민에서 직접 기입하고, 전체 월별 수익을 대시보드에서 조회할 수 있도록 한다.

### 2-2. 가격 정책

| 유형 | 금액 | 비고 |
|---|---|---|
| 정가 | 39,000원 | 기본가 (고정) |
| 할인가 | 직접 입력 | 운영자 재량 적용, 금액 자유 입력 |

### 2-3. 회원 목록 — 추가 컬럼

기존 회원 목록 테이블에 아래 컬럼을 추가한다.

| 컬럼명 | 내용 | 표시 방식 |
|---|---|---|
| 입금 여부 | 입금완료 / 미입금 | 컬러 배지 |
| 입금액 | 금액 + 유형 표시 | `24,000원` + `할인가` 배지 |

**필터 추가:** 입금 여부 (전체 / 입금완료 / 미입금)

### 2-4. 회원 상세 — 입금 정보 섹션

회원 상세 페이지에 독립된 **입금 정보** 패널을 추가한다.

**입력 항목:**

- **가격 유형** (라디오 선택)
  - 정가: 39,000원 고정 표시
  - 할인가: 받은 금액을 숫자로 직접 입력
- **입금일** (날짜 선택)
- **입금 상태** (드롭다운): 입금완료 / 미입금

**저장 시 동작:** 해당 회원 레코드에 저장, 수익 집계에 자동 반영

### 2-5. 입금·수익 현황 페이지 (신규)

사이드바에 **입금·수익 현황** 메뉴를 신설한다.

**상단 지표 카드 (3개):**

| 지표 | 설명 |
|---|---|
| 이달 총 입금액 | 해당 월 전체 입금 합산 |
| 이달 순수익 | 총 입금액 기준 |
| 누적 수익 | 연간 누적 |

**월별 수익 추이 차트:** 연도 선택 → 월별 바 차트

**구성 현황:** 정가 / 할인가 비율 바 형태 표시

**입금 내역 리스트:** 회원명 / 날짜 / 유형 / 금액, 월별 필터

## 3. 기능 2 — 담당매니저 배정

### 3-1. 개요

각 회원에게 담당 매니저를 배정하여, 매니저가 어드민에 로그인했을 때 자신이 담당하는 회원만 조회할 수 있게 한다.

**현재 매니저 (3명):** 김세연 / 이영현 / 송규용

### 3-2. 회원 목록 — 추가 컬럼 및 필터

| 컬럼명 | 내용 |
|---|---|
| 담당매니저 | 배정된 매니저 이름 배지. 미배정 시 `[배정]` 버튼 노출 |

**필터 추가:** 담당매니저 드롭다운 (전체 / 김세연 / 이영현 / 송규용)

매니저 계정으로 로그인 시 기본 필터가 본인 담당 회원으로 고정됨.

### 3-3. 회원 상세 — 담당매니저 배정 UI

회원 정보 패널 하단에 매니저 칩 선택 UI를 배치한다.

- 매니저 목록(김세연 / 이영현 / 송규용)을 칩 형태로 나열
- 선택된 매니저는 강조 표시
- 변경 즉시 저장 (별도 저장 버튼 없음)

### 3-4. 매니저 관리 페이지 (신규)

사이드바에 **매니저 관리** 메뉴를 신설한다.

**조회 항목:**

| 항목 | 설명 |
|---|---|
| 매니저 이름 / 이메일 | 기본 정보 |
| 담당 회원 수 | 현재 배정된 회원 수 |
| 입금 관리 금액 | 담당 회원의 총 입금액 합산 |

**[담당 회원보기] 버튼:** 클릭 시 해당 매니저 필터가 적용된 회원 목록으로 이동

## 4. 기능 3 — 프로필 카드뷰

### 4-1. 개요

온보딩(프로필 작성)을 완료한 회원을 카드 형태로 한 번에 조회하여, 매칭 대상을 시각적으로 비교할 수 있도록 한다. **조회 전용이며, 프로필 전달 및 매칭 진행은 수동으로 별도 처리한다.**

### 4-2. 진입 경로

- 사이드바 **프로필 카드뷰** 메뉴
- 회원 목록 상단 **[프로필 카드뷰]** 버튼

### 4-3. 필터

| 필터 | 옵션 |
|---|---|
| 성별 | 전체 / 남성 / 여성 |
| 나이대 | 전체 / 20대 / 30대 |
| 담당매니저 | 전체 / 김세연 / 이영현 / 송규용 |

### 4-4. 카드 구성

**카드 헤더:**
- 프로필 사진 (없을 경우 이니셜 아바타)
- 성별 배지 (남/여)

**카드 본문:**
- 이름
- 나이 · 지역 · 직업
- 주요 조건 칩: 키 / 학력 / 흡연 여부

**카드 하단 액션:**
- **[상세보기]** → 회원 상세 페이지로 이동

## 5. 기능 4 — 반려 확인 모달

### 5-1. 개요

PENDING 상태 회원의 가입 신청을 반려할 때, 실수로 버튼을 누르는 것을 방지하기 위해 확인 모달을 노출한다.

### 5-2. 노출 위치

- 회원 목록 → PENDING 회원 행의 **[반려]** 버튼 클릭 시
- 대시보드 → 승인 대기 패널의 **[반려]** 버튼 클릭 시

### 5-3. 모달 구성

| 요소 | 내용 |
|---|---|
| 제목 | 반려 처리 확인 |
| 본문 | "{회원명}님의 가입 신청을 반려합니다. 반려 후에는 회원이 재신청해야 합니다." |
| 버튼 | 취소 / 반려 확인 (빨간색) |

**[취소]** 클릭 시: 모달 닫힘, 아무 동작 없음  
**[반려 확인]** 클릭 시: 해당 회원 상태를 REJECTED로 변경

## 6. 정보 구조(IA) 변경사항

```
Clover Admin
├── 대시보드
│   └── 미입금 현황, 승인 대기 회원
│
├── 회원 관리
│   ├── 회원 목록          ← 입금 여부·입금액·담당매니저 컬럼 추가
│   │                        PENDING 회원 승인/반려 버튼 추가
│   │   └── 회원 상세      ← 입금 정보 섹션, 담당매니저 배정 추가
│   └── 프로필 카드뷰      ← 신규 (조회 전용)
│
├── 정산 / 수익
│   └── 입금·수익 현황     ← 신규
│
└── 운영
    ├── 매니저 관리        ← 신규
    └── 설정
```
